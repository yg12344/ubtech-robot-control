# Android APK 打包说明

## 前置要求

### Windows系统
1. 安装Python 3.8+
2. 安装Java JDK 11或更高版本
3. 安装Android SDK和NDK
4. 配置环境变量

### Linux/Mac系统
1. 安装Python 3.8+
2. 安装Java JDK 11或更高版本
3. 安装必要的依赖库

## 安装Buildozer

### 在Windows上使用WSL2（推荐）

```bash
# 在WSL2中安装依赖
sudo apt update
sudo apt install -y build-essential git python3 python3-pip python3-dev \
    openjdk-11-jdk autoconf libtool pkg-config zlib1g-dev libncurses5-dev \
    libncursesw5-dev libtinfo5 cmake libffi-dev libssl-dev

# 安装buildozer
pip3 install buildozer
```

### 在Mac上

```bash
brew install python3
pip3 install buildozer

# 安装依赖
brew install autoconf automake libtool pkg-config
```

## 打包步骤

### 1. 安装依赖

```bash
pip install -r requirements_android.txt
```

### 2. 初始化Buildozer（首次）

```bash
buildozer init
```

这会创建一个默认的 `buildozer.spec` 文件，我们已提供优化版本。

### 3. 修改buildozer.spec

确保以下配置正确：

```ini
title = 优必选小微控制
package.name = ubtech_robot_control
package.domain = org.ubtech
android.api = 33
android.minapi = 21
android.archs = arm64-v8a,armeabi-v7a
```

### 4. 创建资源目录

```bash
mkdir -p assets
```

添加以下文件到 `assets/` 目录：
- `icon.png` - 应用图标（512x512px）
- `presplash.png` - 启动画面（1024x1024px）

### 5. 调试模式打包（快速）

```bash
buildozer android debug
```

首次编译会下载Android NDK、SDK等，需要较长时间（约30-60分钟）。

### 6. 发布模式打包

```bash
buildozer android release
```

## 输出文件

编译完成后，APK文件位于：
```
bin/ubtech_robot_control-1.0.0-arm64-v8a-release.apk
```

## 签名APK（可选）

如需要对APK进行签名：

```bash
# 创建密钥库
keytool -genkey -v -keystore my-key.keystore -alias alias_name \
    -keyalg RSA -keysize 2048 -validity 10000

# 在buildozer.spec中配置签名
buildozer.sign = 1
buildozer.keystore = my-key.keystore
buildozer.keystore_password = your_password
buildozer.key_alias = alias_name
buildozer.key_password = your_password

# 重新打包
buildozer android release
```

## 常见问题

### 1. 编译失败

**问题**: `Error: Unable to find platform-specific libraries for Android`

**解决**:
```bash
buildozer android clean
buildozer android debug
```

### 2. 依赖库安装失败

**问题**: 某些Python库无法在Android上使用

**解决**: 在 `requirements` 中移除不兼容的库，或使用Android兼容的替代品。

### 3. 权限问题

**问题**: 应用崩溃或无法连接

**解决**: 确保 `buildozer.spec` 中的权限配置正确：
```ini
android.permissions = BLUETOOTH, BLUETOOTH_ADMIN, BLUETOOTH_SCAN, BLUETOOTH_CONNECT, INTERNET, RECORD_AUDIO
```

### 4. APK文件过大

**问题**: APK超过100MB

**解决**:
- 只选择必要的架构（如只保留 arm64-v8a）
- 移除不必要的依赖库
- 使用APK分包

## 快速开始脚本

创建 `build_apk.sh`：

```bash
#!/bin/bash

# 清理旧文件
buildozer android clean

# 调试模式打包
buildozer android debug

# 查找生成的APK
find bin -name "*.apk" -exec cp {} . \;

echo "APK已生成到当前目录"
```

使用：
```bash
chmod +x build_apk.sh
./build_apk.sh
```

## 部署到设备

### 方法1: 通过USB安装

```bash
adb install ubtech_robot_control-1.0.0-arm64-v8a-release.apk
```

### 方法2: 通过蓝牙或WiFi传输

将APK文件传输到Android设备，然后打开安装。

## 测试应用

安装后：
1. 打开"优必选小微控制"应用
2. 进入"系统设置"配置连接方式
3. 在主界面点击"连接机器人"
4. 测试各项功能

## 注意事项

1. **首次编译**: 需要下载大量文件，建议在稳定的网络环境下进行
2. **存储空间**: 确保至少有10GB的可用空间
3. **内存**: 编译过程需要大量内存，建议至少8GB
4. **权限**: 应用需要蓝牙、网络、录音等权限，首次启动时需授权

## 优化建议

### 1. 减小APK体积

```ini
# 只保留主要架构
android.archs = arm64-v8a

# 移除不必要的库
requirements = python3,kivy,kivymd,pyjnius,android,plyer
```

### 2. 提高启动速度

```ini
# 启用ART编译
android.accepted_sdk_versions = 33
```

### 3. 优化内存使用

在代码中使用更高效的数据结构和算法。
