# APK生成快速指南

## 重要说明

**APK文件目前不存在，需要使用Buildozer工具进行编译打包。**

## 推荐方案：使用在线服务（最快）

### 方案1: GitHub Actions 自动打包（推荐）

1. **将代码上传到GitHub**
```bash
# 初始化Git仓库
git init
git add .
git commit -m "Initial commit"

# 创建GitHub仓库后，关联远程仓库
git remote add origin https://github.com/你的用户名/ubtech-robot-control.git
git branch -M main
git push -u origin main
```

2. **创建`.github/workflows/build-apk.yml`**
```yaml
name: Build APK

on:
  push:
    branches: [ main ]
  workflow_dispatch:

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - uses: actions/checkout@v3

    - name: Set up Python
      uses: actions/setup-python@v4
      with:
        python-version: '3.9'

    - name: Install dependencies
      run: |
        pip install --upgrade pip
        pip install buildozer cython

    - name: Build APK
      run: |
        buildozer android debug

    - name: Upload APK
      uses: actions/upload-artifact@v3
      with:
        name: release-apk
        path: bin/*.apk
```

3. **触发构建**
   - Push代码到GitHub，自动触发构建
   - 或在GitHub Actions页面手动触发

4. **下载APK**
   - 构建完成后，在Actions页面下载artifact

## 本地打包（需要配置环境）

### Windows系统（推荐使用WSL2）

#### 1. 安装WSL2
```powershell
# 以管理员身份运行PowerShell
wsl --install
# 重启电脑
```

#### 2. 在WSL2中安装依赖
```bash
# 更新系统
sudo apt update
sudo apt upgrade -y

# 安装编译工具
sudo apt install -y build-essential git python3 python3-pip python3-dev \
    openjdk-11-jdk autoconf libtool pkg-config zlib1g-dev libncurses5-dev \
    libncursesw5-dev libtinfo5 cmake libffi-dev libssl-dev

# 验证Java安装
java -version
```

#### 3. 安装Buildozer
```bash
# 升级pip
pip3 install --upgrade pip

# 安装buildozer
pip3 install buildozer

# 验证安装
buildozer --version
```

#### 4. 进入项目目录
```bash
cd /mnt/c/Users/Administrator/CodeBuddy/20260312111217
```

#### 5. 创建资源文件
```bash
# 创建assets目录
mkdir -p assets

# 添加占位图标和启动画面（或使用你自己的图片）
# 可以下载临时图标: https://www.flaticon.com/
# 将icon.png和presplash.png放入assets/目录
```

#### 6. 打包APK
```bash
# 首次打包（需要30-60分钟，下载SDK/NDK）
buildozer android debug

# 生成的APK在: bin/ubtech_robot_control-1.0.0-arm64-v8a-debug.apk
```

## 打包命令详解

### 调试模式（快速，适合测试）
```bash
buildozer android debug
```
- 编译速度快
- 文件较小
- 包含调试信息

### 发布模式（生产用）
```bash
buildozer android release
```
- 编译速度慢
- 文件较大
- 优化性能
- 不包含调试信息

### 清理构建
```bash
buildozer android clean
```

### 更新依赖
```bash
buildozer android update
```

## 常见问题

### Q: 编译失败 "Unable to find platform-specific libraries"
A:
```bash
buildozer android clean
buildozer android debug
```

### Q: 网络太慢，下载SDK/NDK超时
A:
- 使用国内镜像（手动下载SDK/NDK）
- 或使用GitHub Actions云编译

### Q: 没有assets目录怎么办？
A:
```bash
mkdir -p assets
# 创建临时图标（可以使用任何PNG图片）
# 或从网上下载替换
```

### Q: APK文件太大
A:
- 在buildozer.spec中只保留一个架构：`android.archs = arm64-v8a`
- 使用release模式打包
