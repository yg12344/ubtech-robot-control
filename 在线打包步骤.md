# 在线打包APK - 详细步骤

## 🎯 最快方法：使用GitHub Actions自动打包

### 步骤1：创建GitHub仓库

1. **登录GitHub**
   - 访问 https://github.com
   - 登录你的账号

2. **创建新仓库**
   - 点击右上角 `+` 按钮
   - 选择 `New repository`
   - 仓库名称：`ubtech-robot-control`
   - 选择 `Public` 或 `Private`
   - **不要**勾选 "Initialize this repository with a README"
   - 点击 `Create repository`

### 步骤2：上传代码到GitHub

#### 方法A：使用Git命令行（推荐）

**如果是第一次使用Git，先配置用户信息：**
```bash
git config --global user.name "你的用户名"
git config --global user.email "你的邮箱@example.com"
```

**上传代码：**
```bash
# 进入项目目录
cd c:/Users/Administrator/CodeBuddy/20260312111217

# 初始化Git仓库
git init

# 添加所有文件
git add .

# 提交代码
git commit -m "Initial commit - 优必选小微机器人控制软件"

# 关联GitHub仓库（替换YOUR_USERNAME为你的GitHub用户名）
git remote add origin https://github.com/YOUR_USERNAME/ubtech-robot-control.git

# 推送到GitHub
git branch -M main
git push -u origin main
```

**如果需要输入密码，使用：**
- 用户名：你的GitHub用户名
- 密码：使用GitHub Personal Access Token（不是账号密码）

#### 方法B：使用GitHub网页上传（简单但有限制）

1. 在新创建的仓库页面
2. 点击 `uploading an existing file`
3. 拖拽项目文件夹中的所有文件（除了隐藏文件夹）
4. 滚动到底部，点击 `Commit changes`

**注意：这种方法不适合大文件，推荐使用命令行。**

### 步骤3：触发自动打包

代码上传成功后，GitHub Actions会**自动开始打包**。

**查看打包进度：**
1. 进入你的GitHub仓库
2. 点击顶部的 `Actions` 标签
3. 你会看到 "Build Android APK" 工作流正在运行
4. 点击进入可以查看详细日志

**打包时间：**
- 首次打包：约 30-60 分钟
- 后续打包：约 20-30 分钟

### 步骤4：下载APK

**打包完成后：**

1. **进入Actions页面**
   - 点击仓库的 `Actions` 标签

2. **找到成功的构建**
   - 找到绿色的 ✓
   - 点击进入该工作流运行

3. **下载APK**
   - 滚动到页面底部
   - 找到 `Artifacts` 部分
   - 点击 `ubtech-robot-control-debug-apk`
   - 下载ZIP文件

4. **解压ZIP**
   - 解压下载的ZIP文件
   - 里面包含 `.apk` 文件
   - 这个就是你要的APK！

### 步骤5：安装APK到手机

#### 方法A：USB连接安装
```bash
# 启用手机的开发者选项和USB调试
adb install ubtech_robot_control-1.0.0-arm64-v8a-debug.apk
```

#### 方法B：蓝牙或WiFi传输
1. 将APK文件传输到手机
2. 在手机上点击APK文件
3. 允许安装未知来源应用
4. 完成安装

#### 方法C：云存储分享
1. 上传APK到云盘（百度网盘、Google Drive等）
2. 分享链接到手机
3. 下载并安装

## 🚀 手动触发打包（可选）

如果自动打包没有运行，你可以手动触发：

1. 进入仓库的 `Actions` 标签
2. 左侧选择 "Build Android APK"
3. 点击右侧的 `Run workflow` 按钮
4. 选择分支（main）
5. 点击 `Run workflow` 绿色按钮

## 🛠️ 如果打包失败

### 检查日志
1. 点击失败的工作流
2. 展开红色的步骤
3. 查看错误信息

### 常见问题

**问题1：代码上传失败**
```
fatal: remote origin already exists
```
解决：
```bash
git remote remove origin
git remote add origin https://github.com/YOUR_USERNAME/ubtech-robot-control.git
```

**问题2：打包超时**
- 正常现象，首次打包需要30-60分钟
- 耐心等待，GitHub Actions不会超时

**问题3：找不到APK**
- 检查Actions页面是否有错误
- 查看build-logs artifact获取详细信息

如果遇到问题：

1. **查看GitHub Actions日志**

   - 或手动触发workflow
